# Security Policy

## Supported versions

Security updates are provided on the `main` branch.

## Reporting a vulnerability

Please do **not** open a public issue for security vulnerabilities.

Instead:

1. Use GitHub’s “Report a vulnerability” feature (Security Advisories) for this repository, or
2. If you cannot use advisories, open a private report to the repository owner.

Please include a clear description, reproduction steps, and impact assessment. If possible, provide a suggested fix or mitigation.
